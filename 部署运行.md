# KnowledgeOS 部署与运行指南 (v0.5.3)

本文档提供 KnowledgeOS 全栈系统的完整部署方案，包含后端容器化集群部署（支持 Mac/Ubuntu/Windows）及客户端封装打包方法。

---

## 第一部分：后端环境部署 (Backend)

后端采用 Docker 容器化方案，一键拉起 PostgreSQL、Redis、FastAPI 服务及 Celery 异步任务中心。

### 1. 安装 Docker

根据你的操作系统安装 Docker：

- **macOS**:
  - 下载并安装 [Docker Desktop for Mac](https://www.docker.com/products/docker-desktop/)。
  - 建议至少分配 4GB 内存给 Docker（在 Settings -> Resources 中配置）。
- **Windows**:
  - 下载并安装 [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/)。
  - 确保已启用 WSL 2 (Windows Subsystem for Linux)。
- **Ubuntu/Linux**:
  - 运行以下命令：
    ```bash
    sudo apt-get update
    sudo apt-get install docker-ce docker-ce-cli containerd.io docker-compose-plugin
    ```

### 2. 配置环境变量

在启动前，必须准备好模型 API Key。

1. 进入 `server` 目录：
   ```bash
   cd server
   ```
2. 复制环境变量模版：
   ```bash
   cp .env.example .env
   ```
3. 编辑 `.env` 文件，填入你的 API Key（Gemini, DeepSeek 等）。
4. (可选) 修改 `models_config.yaml` 配置文件，定义你想使用的模型路由。

### 3. 一键启动集群

在 `server` 目录下运行：
```bash

conda create -n KOSConda python=3.12 -y
conda activate KOSConda

# pip install -r requirements.txt

docker compose up --build -d
```
Docker 会自动构建镜像并拉起 4 个核心服务：
- `kos_api`: 提供 Restful API 接口 (端口 8080)
- `kos_celery_worker`: 处理文档解析、爬虫与图谱生成
- `kos_postgres`: 持久化存储用户、项目与知识节点
- `kos_redis`: 任务队列中心与 SSE 日志中转

### 4. 初始化数据库 (仅首次运行)

集群启动后，需要初始化数据库表结构：
```bash
docker exec -it kos_api python init_db.py

docker exec kos_postgres psql -U kos_admin -d knowledgeos -f /app/migrate_platform_cookies.sql

预制的 system-admin 用户的登录凭据如下：

用户名 (Username): system-admin
密码 (Password): SystemPass123!
```

## 第二部分：客户端开发与打包 (Client)

客户端基于 Tauri + React 构建，支持封装为 Mac 和 Windows 的原生安装程序。

### 1. 安装开发依赖

你需要在本地机器（非 Docker）安装以下环境：
- **Node.js**: 建议 v18+
- **Rust**: [安装 Rust 环境](https://www.rust-lang.org/tools/install) (Tauri 编译需要)

### 2. 运行开发版

```bash
cd client
npm install
npm run tauri dev
```

### 3. 封装打包 (Production Build)

#### macOS 平台 (.dmg / .app)
在 Mac 上运行：
```bash
npm run tauri build
```
- 打包产物位于：`client/src-tauri/target/release/bundle/dmg/`

#### Windows 平台 (.msi / .exe)
在 Windows 上运行：
```bash
npm run tauri build
```
- 打包产物位于：`client/src-tauri/target/release/bundle/msi/`

> **注意**：Tauri 打包是跨系统的，你需要在对应的操作系统上执行打包命令，或者配置 GitHub Actions 进行云端构建。

## 第三部分：日常维护与常见问题


docker compose down 

### 1. 修改配置后如何重载？
如果你修改了 `server/.env` 或 `models_config.yaml`，无需重新构建，只需重启服务：
```bash
docker compose restart api worker
```

### 2. 查看解析日志
如果文档处理卡住了，可以查看 Worker 日志：
```bash
docker logs -f kos_celery_worker
```

### 3. 重置向量数据库
在进行大规模架构调整后，建议清理向量索引：
```bash
rm -rf server/data/lancedb/*
docker compose restart api
```
