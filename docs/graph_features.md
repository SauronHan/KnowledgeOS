# KnowledgeOS 知识图谱功能指南

本文档介绍了 KnowledgeOS 知识图谱顶部的三个核心观察维度：**Type (类型)**、**Community (社区)** 和 **Insights (洞察)**。

---

## 1. Type (类型)
**定义**：基于节点元数据的属性分类视图。

*   **核心逻辑**：系统根据节点的来源和属性（如 `entity`, `concept`, `source`, `query` 等）为节点着色。
*   **颜色指南**：
    *   **Source (来源)**：通常为橙色，代表上传的原始 PDF、Markdown 或网页抓取文件。
    *   **Concept (概念)**：通常为紫色，代表从资料中提炼出的核心知识点或抽象定义。
    *   **Entity (实体)**：蓝色，代表具体的人物、地点、组织等具象对象。
    *   **Query/Research (研究)**：绿色，代表用户的搜索任务或深度研究路径。
*   **应用场景**：用于快速区分知识库的资产构成，了解哪些是原始证据，哪些是提炼后的知识。

## 2. Community (社区/聚类)
**定义**：基于网络拓扑结构的自动聚类视图。

*   **核心逻辑**：使用 **Louvain 算法** 自动计算节点间的连接密度。联系紧密（互相引用多、共现频率高）的节点会被归为同一个“社区”。
*   **核心价值**：**自动发现隐藏主题**。系统不依赖人工标签，而是通过数学模型发现知识的自然分布。
*   **特征分析**：
    *   同色节点代表一个“知识群落”。
    *   节点越大，代表其在社区中的“枢纽”作用越强（引用数/链接数越高）。
*   **应用场景**：用于识别知识库中自然形成的学科领域或研究专题，发现不同知识块之间的边界。

## 3. Insights (洞察)
**定义**：基于算法的主动式研究引导工具。

开启该模式后，系统会通过侧边栏提供以下维度的智能分析：

### A. 意外关联 (Surprising Connections)
*   **特征**：发现跨越了不同“社区”边界或跨越了遥远“类型”的连接。
*   **价值**：这些连接通常代表跨学科的灵感或未被注意到的关联，是创新的源泉。

### B. 知识盲区 (Knowledge Gaps)
*   **孤岛节点 (Isolated Nodes)**：链接数极少的页面。提醒你需要增加 [[wikilinks]] 或进行进一步研究来充实内容。
*   **稀疏聚类 (Sparse Clusters)**：成员不少但内部联系极其脆弱的社区。暗示该领域研究流于表面，需要深入挖掘。
*   **桥接节点 (Bridge Nodes)**：连接了 3 个以上独立社区的关键节点。它们是知识库的“咽喉”，理解这些节点是打通整个知识体系的关键。

---

## 总结建议
*   当你需要**分类检索**时，使用 **Type**。
*   当你需要**宏观预览主题分布**时，使用 **Community**。
*   当你需要**寻找研究切入点或优化知识结构**时，使用 **Insights**。
